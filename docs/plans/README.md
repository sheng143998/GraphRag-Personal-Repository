# 计划文档目录

复杂功能开始实现前，先在本目录编写计划文档。计划文档用于让后续开发可以恢复上下文，也方便用户在开发前确认范围。

## 计划文档需要覆盖的内容

- 要解决的问题。
- 调研过程中发现的重要信息。
- 当前背景。
- 涉及模块。
- 预计修改文件。
- 实现策略。
- 重点审查文件。
- 测试计划。
- 已知风险。

## 命名建议

```text
YYYY-MM-DD-task-name.md
```

示例：

```text
2026-05-26-basic-rag-pipeline.md
```

## 编写约定

- 默认使用中文。
- 不需要写详细时间线。
- 不需要写回滚计划。
- 如果实现过程中发现计划不准确，需要更新计划或在最终交接说明中记录偏差原因。
