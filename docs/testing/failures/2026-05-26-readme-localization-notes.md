## 风险 1：第三方依赖 README 不应修改

- 现象：搜索 README 时发现 `frontend/node_modules/` 下存在大量第三方 README。
- 触发场景：执行 README 文件盘点。
- 根因：前端依赖已安装，依赖包自带文档会被搜索命中。
- 处理方式：本次只修改项目自有 README，明确排除 `frontend/node_modules/**`。
- 后续避免：搜索项目文档时使用排除规则，避免误改第三方依赖。
- 是否补充自动化测试：不涉及代码测试，已通过文件搜索确认自有 README 范围。

## 风险 2：说明性英文和技术标识不能用同一规则处理

- 现象：用户要求“英文都替换为中文”，但 README 中包含命令、路径、接口、环境变量和技术名词。
- 触发场景：翻译 README 内容。
- 根因：命令、路径、接口和配置项如果翻译成中文，会导致无法执行、无法搜索或与代码不一致。
- 处理方式：说明性文字中文化；命令、路径、接口、配置项、框架名保留原文并使用代码格式。
- 后续避免：后续文档中文化任务都采用同一规则：解释文字中文化，技术标识保持准确。
- 是否补充自动化测试：不涉及代码测试，已通过文本检查确认 README 文件存在。
